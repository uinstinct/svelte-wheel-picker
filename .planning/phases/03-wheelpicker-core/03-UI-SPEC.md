## Structural Contract (Component — not consumer styling)

This section is the primary output of Phase 3 UI research. It defines what DOM structure and data attributes the component must emit. All executor decisions about element placement must satisfy this contract.

### DOM Element Map

| Element | Tag | Data Attribute | classNames Key | Required |
|---------|-----|---------------|----------------|----------|
| Outer container | `div` | `data-swp-wrapper` | `wrapper` | yes |
| Option row (each) | `div` | `data-swp-option` | `option` | yes |
| Option text span | `span` | `data-swp-option-text` | `optionText` | yes |
| Selection overlay | `div` | `data-swp-selection` | `selection` | yes |

Source: CONTEXT.md D-03, D-09

### State Data Attributes

| Attribute | Value | Applied To | Meaning |
|-----------|-------|-----------|---------|
| `data-swp-selected` | `"true"` | Currently selected option row (`data-swp-option`) | Option is the active selection |
| `data-swp-disabled` | `"true"` | Disabled option row (`data-swp-option`) | Option is not selectable |

Source: CONTEXT.md D-04

**Rule:** Absent attribute = false state. Never emit `data-swp-selected="false"` or `data-swp-disabled="false"`. Only emit when `true`.

### Selection Overlay Placement

The `data-swp-selection` div is absolutely positioned within `data-swp-wrapper`. It sits above the options list in z-order (pointer-events: none) so it does not interfere with drag interactions. Exact vertical centering uses the `optionItemHeight` prop value. Placement decision is within Claude's discretion (CONTEXT.md), but the element must exist in the DOM regardless of placement strategy.

### classNames Injection Contract

When a consumer passes `classNames={{ wrapper: "my-class" }}`, the class is appended to the element alongside its `data-swp-*` attribute. Both always coexist on the same element. Order: native element classes (none, since headless) → classNames value.

## Interaction States

The executor must ensure all these states are observable via CSS attribute selectors. No JS callbacks are needed for styling — CSS alone must be sufficient.

| State | CSS Selector | Description |
|-------|-------------|-------------|
| Selected option | `[data-swp-option][data-swp-selected="true"]` | Center/active item |
| Disabled option | `[data-swp-option][data-swp-disabled="true"]` | Non-interactive item |
| Wrapper (always) | `[data-swp-wrapper]` | Outer container for sizing/overflow |
| Selection highlight | `[data-swp-selection]` | Center row indicator bar |

## Props Contract (STYL-04)

These props affect layout dimensions and interaction feel. The executor must apply them to the DOM directly as inline style values or CSS custom properties — never hardcoded.

| Prop | Default | Type | Applied To | Notes |
|------|---------|------|-----------|-------|
| `visibleCount` | `5` | `number` (odd only) | Wrapper height = `visibleCount * optionItemHeight` | Must be odd; D-06, D-07 |
| `optionItemHeight` | `40` (match React v1.2.2) | `number` (px) | Each `data-swp-option` row height | Sets the snap grid unit |
| `dragSensitivity` | Copy from React v1.2.2 | `number` | Pointer delta multiplier in RAF loop | Executor reads React source |
| `scrollSensitivity` | Copy from React v1.2.2 | `number` | Wheel event delta multiplier | Executor reads React source |

## Spacing Scale (Demo Site Only)

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between demo labels and wheel |
| sm | 8px | Compact padding inside demo containers |
| md | 16px | Default padding for demo page sections |
| lg | 24px | Spacing between multiple wheel examples |
| xl | 32px | Page-level horizontal padding |
| 2xl | 48px | Major demo section breaks |
| 3xl | 64px | Not used in Phase 3 demo |

Exceptions: Touch targets for the wheel itself are governed by `optionItemHeight` (default 40px, meeting the 40px minimum interactive target guideline). No other exceptions.

## Color (Demo Site Only)

The library ships no color. The demo site uses a minimal neutral palette so the wheel component is the visual focus. Colors are applied exclusively via `data-swp-*` selectors in the demo's CSS.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (light) / `#0f0f0f` (dark) | Page background |
| Secondary (30%) | `#f4f4f5` (light) / `#1c1c1e` (dark) | Demo card/container backgrounds |
| Accent (10%) | `#3b82f6` | Selection highlight overlay (`data-swp-selection`) only |
| Destructive | N/A | No destructive actions in Phase 3 |

Accent reserved for: `[data-swp-selection]` background only. No other element uses the accent color in Phase 3.

**Note:** The demo applies dark mode via `prefers-color-scheme` media query. The component itself is colorless — the consumer provides all color via CSS.

## Copywriting Contract

Phase 3 is a headless component — no user-facing UI copy exists inside the component. Copywriting applies to two surfaces: (1) developer console warnings and (2) the demo site.

| Element | Copy |
|---------|------|

| Demo page heading | `Wheel Picker` |
| Demo section: basic usage | `Single Wheel` |
| Demo section: with disabled options | `Disabled Options` |
| Empty wheel (zero options) | Renders empty container with correct height — no visible text. Consumer is responsible for empty state. |
| Error: invalid prop (TypeScript catches at compile time) | No runtime error copy — TypeScript enforcement only |
| Destructive confirmation | N/A — no destructive actions in Phase 3 |
