**Step 2: Create src/lib/wheel-physics-utils.ts with these pure functions:**

Export the following constants (per D-02, copied from React v1.2.2):
```typescript
export const RESISTANCE = 0.3;
export const MAX_VELOCITY = 30;

export const DEFAULT_ITEM_HEIGHT = 30;
export const DEFAULT_VISIBLE_COUNT = 5;
export const SNAP_BACK_DECELERATION = 10;
```

Export these pure functions:

1. `easeOutCubic(p: number): number` — `Math.pow(p - 1, 3) + 1` (from React source)

2. `indexToOffset(index: number, itemHeight: number, visibleCount: number): number` — converts a selected index to the translateY offset value. Formula: `Math.floor(visibleCount / 2) * itemHeight - index * itemHeight`

3. `offsetToIndex(offset: number, itemHeight: number, visibleCount: number): number` — inverse of indexToOffset. Formula: `Math.round((Math.floor(visibleCount / 2) * itemHeight - offset) / itemHeight)`

4. `clampIndex(index: number, optionsLength: number): number` — `Math.max(0, Math.min(index, optionsLength - 1))`

5. `snapToNearestEnabled(targetIndex: number, options: Array<{ disabled?: boolean }>): number` — if target is not disabled, return it. Otherwise walk outward in both directions (lower-delta first, then upper-delta) to find nearest enabled. If all disabled, return targetIndex.

6. `calculateVelocity(yList: Array<[number, number]>, itemHeight: number): number` — takes the last two entries from yList (each is `[clientY, timestamp]`), computes `((y2 - y1) / itemHeight) * 1000 / (t2 - t1)`, clamps absolute value to `MAX_VELOCITY`, preserves sign. Returns 0 if fewer than 2 entries.

7. `computeSnapTarget(currentIndexFromOffset: number, velocity: number, dragSensitivity: number): number` — computes inertia overshoot target: `baseDeceleration = dragSensitivity * 10`, `overshoot = 0.5 * velocity * velocity / baseDeceleration`, `rawTarget = currentIndexFromOffset + Math.sign(velocity) * overshoot`, returns `Math.round(rawTarget)`.

8. `computeAnimationDuration(distance: number, scrollSensitivity: number): number` — `Math.sqrt(Math.abs(distance) / scrollSensitivity)` (seconds). Minimum 0.1s, maximum 0.6s.

Create `src/lib/use-wheel-physics.svelte.ts` exporting a `WheelPhysics` class.

**Class design:**

```typescript
import type { WheelPickerOption } from './types.js';
import {
  easeOutCubic, indexToOffset, offsetToIndex, clampIndex,
  snapToNearestEnabled, calculateVelocity, computeSnapTarget,
  computeAnimationDuration, RESISTANCE, DEFAULT_DRAG_SENSITIVITY,
  DEFAULT_SCROLL_SENSITIVITY, DEFAULT_ITEM_HEIGHT, DEFAULT_VISIBLE_COUNT,
  SNAP_BACK_DECELERATION
} from './wheel-physics-utils.js';
```

**Public reactive state (must be `$state`):**
- `offset: number` — the current translateY value. This is the ONLY `$state` field. It drives the CSS transform binding in the component.

**Private non-reactive fields (plain class fields, NOT `$state` — per Pitfall 2):**
- `#rafId: number | null` — current RAF handle
- `#isDragging: boolean` — true between pointerdown and pointerup
- `#dragStartOffset: number` — offset value when drag began
- `#yList: Array<[number, number]>` — last 5 pointer positions for velocity calc
- `#lastWheelTime: number` — timestamp of last wheel event (100ms debounce per Pitfall 6)
- `#animating: boolean` — true during snap/inertia animation

**Constructor parameters:**
```typescript
constructor(opts: {
  itemHeight: number;
  visibleCount: number;
  dragSensitivity: number;
  scrollSensitivity: number;
  options: WheelPickerOption[];
  initialIndex: number;
  onSnap: (index: number) => void; // called when snap animation completes
})
```

Store all opts as private fields. Set initial `offset = indexToOffset(initialIndex, itemHeight, visibleCount)`.

**Public methods:**

1. `startDrag(clientY: number): void` — cancel any running RAF, set `#isDragging = true`, record `#dragStartOffset = this.offset`, init `#yList = [[clientY, performance.now()]]`.

2. `moveDrag(clientY: number, dragSensitivity: number): void` — if not dragging, return. Compute `delta = (clientY - this.#yList[0][0]) * dragSensitivity`. Apply resistance if offset would go beyond bounds (offset > indexToOffset(0, ...) or offset < indexToOffset(lastIndex, ...)): `delta *= RESISTANCE`. Set `this.offset = this.#dragStartOffset + delta`. Push `[clientY, performance.now()]` to `#yList`, keep last 5.

   Wait — simpler approach matching React: track the raw delta from the START position, not cumulative. `#dragStartY` captures `clientY` at pointerdown. On each move: `rawDelta = clientY - #dragStartY`. Apply sensitivity: `delta = rawDelta`. The sensitivity is already baked into the inertia calculation, not the drag itself. Actually, looking at React source more carefully: drag sensitivity multiplies the pointer delta directly. So: `delta = (clientY - #dragStartY)`. Then `this.offset = #dragStartOffset + delta`. Keep it simple — sensitivity applies to the VELOCITY calculation for inertia, not to the raw drag offset. Correct that — per the React source, `dragSensitivity` is used in the deceleration calculation (`baseDeceleration = dragSensitivity * 10`), not as a drag multiplier. The raw pointer delta maps 1:1 to offset pixels during drag.

   Revised: `moveDrag(clientY: number): void` — compute `delta = clientY - #dragStartY`. Apply boundary resistance: if resulting offset is out of bounds, multiply the out-of-bounds portion by `RESISTANCE`. Set `this.offset = #dragStartOffset + delta` (with resistance applied). Push `[clientY, performance.now()]` to `#yList`, keep last 5.

3. `endDrag(options: WheelPickerOption[], itemHeight: number, visibleCount: number, dragSensitivity: number, scrollSensitivity: number): void` — set `#isDragging = false`. Compute velocity via `calculateVelocity(#yList, itemHeight)`. If velocity is near zero (Math.abs < 0.5), snap to nearest enabled from current offset position. Otherwise compute snap target via `computeSnapTarget(...)`, clamp to [0, options.length-1], run `snapToNearestEnabled`, then animate to that index.

4. `handleWheel(deltaY: number, options: WheelPickerOption[], itemHeight: number, visibleCount: number, scrollSensitivity: number): void` — 100ms debounce: if `performance.now() - #lastWheelTime < 100`, return. Update `#lastWheelTime`. Compute direction: `deltaY > 0` means scroll down (next item), `deltaY < 0` means scroll up (prev item). Compute current index from offset, move by 1 in that direction, skip disabled, clamp, animate to target.

5. `animateTo(targetIndex: number, itemHeight: number, visibleCount: number, scrollSensitivity: number): void` — cancel existing RAF. Compute target offset via `indexToOffset`. Compute duration via `computeAnimationDuration(distance, scrollSensitivity)`. Run RAF loop: on each frame compute `elapsed / (duration * 1000)`, apply `easeOutCubic`, interpolate offset. On completion: set `this.offset` to exact target, call `onSnap(targetIndex)`.

6. `jumpTo(index: number, itemHeight: number, visibleCount: number): void` — cancel RAF, set `this.offset = indexToOffset(index, itemHeight, visibleCount)` immediately. No animation. Used for initial render.

7. `cancelAnimation(): void` — cancel RAF if running, set `#animating = false`.

8. `destroy(): void` — call `cancelAnimation()`.

**Boundary resistance logic (in moveDrag):**
The valid offset range is `[indexToOffset(lastEnabledIndex, ...), indexToOffset(0, ...)]` (note: offset for index 0 is the LARGEST positive offset, offset for last index is the most negative). If drag pushes offset beyond either bound, apply `RESISTANCE` multiplier to the excess:

```typescript
const maxOffset = indexToOffset(0, itemHeight, visibleCount);
const minOffset = indexToOffset(options.length - 1, itemHeight, visibleCount);
let newOffset = #dragStartOffset + delta;
if (newOffset > maxOffset) {
  newOffset = maxOffset + (newOffset - maxOffset) * RESISTANCE;
} else if (newOffset < minOffset) {
  newOffset = minOffset + (newOffset - minOffset) * RESISTANCE;
}
this.offset = newOffset;
```

**Critical: `offset` is `$state`, but the RAF tick function sets it imperatively. Do NOT read `this.offset` as a dependency inside any `$effect`. The RAF loop is started/stopped by method calls, not by reactive effects.**
