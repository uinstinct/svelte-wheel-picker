# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: shadcn Registry and Demo Site.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — manual CSS custom properties |
| Preset | not applicable |
| Component library | none |
| Icon library | none |
| Font | system-ui, -apple-system, sans-serif (established in +page.svelte) |

**Note (source: +page.svelte):** The project uses no Tailwind, no component library, and no shadcn consumer preset. All styles are manual CSS custom properties declared at `:root`. This project IS a shadcn-svelte registry provider — the shadcn initialization gate is not applicable.

**Note on dark mode (source: +page.svelte):** All color tokens have a light value at `:root` and a dark override inside `@media (prefers-color-scheme: dark)`. The hero section must follow this exact pattern.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Label/value gap in section headers |
| sm | 8px | Paragraph bottom margin, compact inline padding |
| md | 16px | Wheel container padding, element spacing |
| lg | 24px | Section bottom margin, hero internal spacing |
| xl | 32px | Body padding (established at `:global(body)`) |
| 2xl | 48px | Hero section vertical padding (top/bottom) |
| 3xl | 64px | Hero section total min-height target for visual weight |

Exceptions: none

**Source:** Derived from established spacing in +page.svelte (`margin-bottom: 24px` on sections, `padding: 32px` on body, `padding: 16px` on `.wheel-container`).

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / option labels | 16px | 400 | 1.5 |
| Label / muted copy | 14px | 400 | 1.5 |
| Heading (section h2) | 16px | 600 | 1.2 |
| Display (hero title) | 28px | 600 | 1.2 |

**Source (source: +page.svelte):** `h1` was 20px/600/1.2 and `h2` 16px/600; `p` 14px/400; wheel options 16px/400. The hero title (`svelte-wheel-picker`) is upgraded to 28px/600 to establish clear landing-page hierarchy — discretion area from CONTEXT.md.

The hero one-line description uses 16px/400/1.5 (body role). The code snippet lines (`npm install ...`, `npx shadcn-svelte ...`) use 14px/400 in a monospace stack: `ui-monospace, SFMono-Regular, Menlo, monospace`.

## Color

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `#ffffff` | `#0f0f0f` | Page background, surfaces |
| Secondary (30%) | `#f4f4f5` | `#1c1c1e` | Wheel containers, hero code block background |
| Text primary | `#111111` | `#eeeeee` | All headings and body text |
| Text muted | `#555555` | `#aaaaaa` | Paragraph copy, selected value readout, description copy |
| Accent (10%) | `rgba(59,130,246,0.15)` bg / `rgba(59,130,246,0.3)` border | same | Selection highlight overlay only |

**Source (source: +page.svelte):** CSS custom properties `--color-bg`, `--color-text`, `--color-text-muted`, `--color-surface` are already established. The accent blue is used exclusively for `[data-swp-selection]` highlight — it must not be introduced on any hero element.

Accent reserved for: **selection highlight overlay on `[data-swp-selection]` only.** Do not use accent blue on hero buttons, badges, code blocks, or links.

Destructive: not applicable — this phase has no destructive actions.

**Hero code block:** Use `--color-surface` as background with a `border-radius: 6px` and `padding: 8px 12px`. Do not introduce new color tokens. Use `--color-text-muted` for the code text.

## Interaction Contract

| Interaction | Behaviour |
|-------------|-----------|
| Code snippet click-to-copy | Not required for this phase — plain `<pre>/<code>` only. No JS copy button. |
| Wheel interactions | All four existing examples remain fully interactive (drag, scroll, keyboard, touch). |
| Dark mode | Hero inherits existing `prefers-color-scheme: dark` media query via CSS custom properties. No additional JS theming. |
| Mobile viewport | `<main>` max-width 400px centred; hero content wraps naturally — no horizontal scrolling. Install block wraps gracefully if the command string exceeds container width (use `overflow-x: auto` on `<pre>`). |
| Hover states | None required on hero — no interactive hero elements. |
