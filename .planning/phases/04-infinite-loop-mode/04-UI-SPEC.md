## Interaction Contract

These contracts govern the observable behavior of the `infinite` feature. The executor must implement all of them:

### Ghost Item DOM Contract (from CONTEXT.md D-01, D-03)

| Attribute | Real items | Ghost items |
|-----------|-----------|-------------|
| `data-swp-option` | present | present |
| `data-swp-option-text` | present | present |
| `data-swp-selected="true"` | present on selected item | NEVER present |
| `data-swp-disabled="true"` | present when disabled | present when disabled |

### Boundary Behavior Contract (from CONTEXT.md D-04, D-05, D-06, D-07, D-08)

| Trigger | `infinite=false` | `infinite=true` |
|---------|-----------------|-----------------|
| Drag past last item | Rubber-band resistance, snap back | No resistance; ghost of first item visible |
| Drag past first item | Rubber-band resistance, snap back | No resistance; ghost of last item visible |
| Release after inertia | Snap to nearest enabled item; stop at boundary | Snap to nearest enabled item; wrap seamlessly |
| ArrowDown on last item | No-op | Animate 1 step forward to ghost of item 0, offset normalizes silently |
| ArrowUp on first item | No-op | Animate 1 step backward to ghost of last item, offset normalizes silently |
| Home / End | Jump to first/last enabled item | Jump to first/last enabled item (same behavior — absolute navigation) |
| Mouse wheel past boundary | Stop at boundary | Wrap to opposite end via ghost |

### Offset Normalization Contract (from CONTEXT.md D-04)

- Normalization happens exactly once per snap settle, inside the `onSnap` callback.
- Normalization must NOT run during active drag or active inertia animation.
- After normalization, `physics.currentIndex` returns a value in `[0, N-1]` — never a ghost index.
- No visual movement is produced by normalization (it happens at rest, at the same pixel position mod options-height).

### Finite Mode Regression Contract

- All Phase 3 behavior remains unchanged when `infinite` prop is absent or `false`.
- Rubber-band resistance, boundary clamping, keyboard no-ops at ends — all must work identically.
- Existing tests must pass without modification.
