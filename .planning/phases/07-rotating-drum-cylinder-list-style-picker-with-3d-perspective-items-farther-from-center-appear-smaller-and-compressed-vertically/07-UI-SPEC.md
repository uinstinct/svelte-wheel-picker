## 3D Visual Contract

This section is specific to Phase 7 and documents the faux-3D rendering contract for the executor.

### Transform Formula

```
cylindricalScaleY(slotIndex, offset, itemHeight, visibleCount):
  dist  = slotIndex + offset / itemHeight - floor(visibleCount / 2)
  angle = dist * PI / visibleCount
  return max(0.1, cos(angle))
```

Source: RESEARCH.md Pattern 1 — derived from cosine projection, cross-verified numerically.

### Per-Item Style Application

| CSS property | When `cylindrical={true}` | When `cylindrical={false}` |
|---|---|---|
| `style:transform` | `scaleY({scale})` where scale = cylindricalScaleY result | `undefined` (attribute omitted) |
| `style:opacity` | `{scale}` (same value — single curve for both depth cues) | `undefined` (attribute omitted) |

**Caching:** Use `{@const scale = ...}` inside each `{#each}` block to avoid calling `cylindricalScaleY` twice per item per frame.

### Slot Index Assignments

| Block | Variable | slotIndex expression |
|-------|----------|----------------------|
| Before-ghosts (infinite mode) | `g` (0-indexed) | `g - options.length` |
| Real items | `i` (0-indexed) | `i` |
| After-ghosts (infinite mode) | `j` (0-indexed) | `options.length + j` |

### Scale Boundaries

- Center item (dist = 0): scaleY = 1.0, opacity = 1.0 — full size, fully opaque
- Edge items at floor(visibleCount/2) slots from center: scaleY ≈ cos(PI/2) = 0 → clamped to 0.1
- Items beyond visible window during fast scroll: clamped to MIN_CYLINDRICAL_SCALE = 0.1

### Wrapper Attribute

`data-swp-cylindrical='true'` is added to the wrapper `<div>` when `cylindrical={true}`, `undefined` when `false`. Follows established `undefined`-not-`false` pattern.

### Demo CSS for Cylindrical Section

The new demo section reuses all existing CSS classes unchanged:
- `.wheel` on `data-swp-wrapper` — 200px width, cursor grab, user-select none, touch-action none
- `.wheel-selection` on `data-swp-selection` — blue tint highlight band (rgba(59,130,246,0.15) bg)
- `.wheel-option` on `data-swp-option` — 16px/400, transition opacity 0.15s

**No new CSS classes are needed for the demo.** The `scaleY` transform and `opacity` are applied as inline styles by the component when `cylindrical={true}`. The existing `.wheel-option` `transition: opacity 0.15s` rule will animate the opacity changes — this is acceptable and intentional.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Drag (touch/mouse) | Unchanged — physics unaffected. `scaleY` reacts to `physics.offset` on every animation frame. |
| Scroll wheel | Unchanged. |
| Keyboard navigation | Unchanged. |
| `cylindrical={true}` toggle | Immediately applies `scaleY` + `opacity` inline styles to all items; no transition needed on prop change. |
| `cylindrical={false}` (default) | No transform, no opacity inline style — byte-for-byte identical to current flat behavior. |
| Infinite mode + cylindrical | Before-ghost and after-ghost items receive the same `cylindricalScaleY` transform with their corrected slot indices. Visual continuity at wrap boundary is maintained. |

## Accessibility Contract

| Concern | Decision |
|---------|----------|
| scaleY on option items | Items remain in DOM and focusable even at min scale (0.1). No items are hidden via `display:none` or `visibility:hidden`. |
| opacity falloff | Items at min opacity (0.1) are still technically visible — not removed from accessibility tree. |
| `data-swp-cylindrical` attribute | Informational only — no aria impact. |
| Keyboard users | All keyboard navigation unchanged. scaleY is purely visual. |
