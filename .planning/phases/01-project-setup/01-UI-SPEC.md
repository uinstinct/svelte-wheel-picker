# Phase 1 — UI Design Contract

> Visual and interaction contract for the Phase 1 project scaffold.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Scope Note

Phase 1 is a toolchain and scaffold phase. It delivers zero user-facing component UI. The only visible output is:

1. A placeholder `WheelPicker.svelte` smoke-test component — a single `<div>` with a text label, used only to confirm the import chain works.
2. A demo route `src/routes/+page.svelte` — renders the placeholder component on screen, used only to verify no TypeScript errors occur.

Neither item has a design specification. They are intentionally minimal and will be replaced entirely in Phase 3. The design contract below governs these placeholders and establishes baseline tokens for all subsequent phases.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none

Note: Phase 1 placeholder UI uses no spacing tokens. These values are declared here as the baseline for all subsequent phases, particularly Phase 3 (WheelPicker Core) and Phase 6 (Demo Site). Executor must not apply spacing to the placeholder `<div>` beyond browser defaults.

Source: Standard 8-point scale default. No upstream artifact specified deviations.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff | Background surfaces |
| Secondary (30%) | #f4f4f5 | Cards, side panels |
| Accent (10%) | #18181b | Selection highlight, interactive focus rings |
| Destructive | #dc2626 | Destructive actions only (none in Phase 1) |

Accent reserved for: selection highlight overlay (center row indicator in Phase 3), focus rings on keyboard-navigable elements, active state of the picker wheel.

Note: Phase 1 applies no color. These are baseline tokens declared for downstream phases. The library is headless — consumers apply color via their own CSS targeting `data-rwp-*` attributes. The tokens above apply only to the demo site (Phase 6).

Source: CLAUDE.md ("Headless — no CSS shipped, data attributes for targeting"). Color values are standard defaults.

## Interaction Contract

Phase 1 has no interactive elements. The placeholder `WheelPicker.svelte` is a static `<div>` with text. No pointer events, keyboard handlers, or scroll behavior are implemented. All interaction requirements (INTR-01 through INTR-06) are deferred to Phases 2–4.

The sole data attribute in Phase 1 is `data-wheel-picker` on the root `<div>` of the placeholder. This establishes the `data-rwp-*` naming pattern but does not carry styling significance in Phase 1.

Source: RESEARCH.md Pattern 6 (placeholder component structure), REQUIREMENTS.md (all INTR-* requirements mapped to Phase 3).
