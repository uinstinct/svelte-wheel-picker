**WheelPicker.svelte — three changes:**

1. **Extract onSnap into a named function** (avoids duplicating the closure body). Define `handleSnap` in the component script scope, after the `initialIndex` derived but before the `physics` constructor:

```typescript
function handleSnap(index: number) {
  if (infinite) {
    const wrappedIndex = wrapIndex(index, options.length);
    physics.jumpTo(wrappedIndex);
    const opt = options[wrappedIndex];
    if (opt && !opt.disabled) {
      state.current = opt.value;
    }
  } else {
    const opt = options[index];
    if (opt && !opt.disabled) {
      state.current = opt.value;
    }
  }
}
```

Then change the `onSnap` in the WheelPhysics constructor (line 67-91) to just `onSnap: handleSnap`.

2. **Add a `$effect` after the physics constructor** (after line 92) that syncs all props to the physics engine when they change:

```typescript
// Sync prop changes to physics engine — fixes state_referenced_locally warnings
$effect(() => {
  physics.update({
    itemHeight: optionItemHeight,
    visibleCount,
    dragSensitivity,
    scrollSensitivity,
    infinite,
    options,
    onSnap: handleSnap,
  });
});
```

This works because `$effect` reads the destructured props reactively. When any prop changes, the effect re-runs and pushes the new values into the physics engine via the existing `update()` method. The `handleSnap` function is re-passed each time so its closure captures current `infinite` and `options` values. The initial run is harmless since `update()` sets the same values the constructor already set.
