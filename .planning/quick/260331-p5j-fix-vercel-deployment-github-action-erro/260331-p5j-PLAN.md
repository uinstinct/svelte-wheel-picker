The error "Git author must have access to the team on Vercel" occurs because Vercel's
deployment is associating the git commit author with team membership. On Hobby plans,
collaboration is not supported.

There are two things to fix:

1. **Disable Vercel's automatic GitHub integration deployments** so only our CLI workflow
   deploys. This is done by adding `vercel.json` with `"github": { "enabled": false }` OR
   by going to Vercel dashboard. Since we cannot access the dashboard, add a `vercel.json`
   at the project root with:
   ```json
   {
     "$schema": "https://openapi.vercel.sh/vercel.json",
     "github": {
       "enabled": false
     }
   }
   ```
   This prevents the Vercel GitHub integration from auto-deploying on push (which is where
   the git-author check happens). Our CLI workflow will handle all deployments.

2. **Ensure the deploy.yml workflow is robust**: The existing workflow already uses
   `--token` flags correctly. No changes needed to deploy.yml itself -- the issue is the
   parallel Vercel GitHub integration triggering its own deployment with git-author checks.

Files to create:
- `vercel.json` (new file at project root)

Files to verify (no changes needed):
- `.github/workflows/deploy.yml` (already correct)
