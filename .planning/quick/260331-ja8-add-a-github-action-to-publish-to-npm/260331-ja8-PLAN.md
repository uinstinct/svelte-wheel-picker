  <files>.github/workflows/publish.yml, package.json</files>
  <action>
    **package.json** — add `publishConfig` field so `--access public` is the default for this scoped package (not just a CI flag). Insert after the `"sideEffects": false` line:

    ```json
    "publishConfig": {
      "access": "public"
    },
    ```

    **publish.yml** — create `.github/workflows/publish.yml` with the following:

    - Trigger: `on: release: types: [published]` (fires only when a draft release is published, not on tag push)
    - Top-level `permissions: {}` (deny all by default; job overrides only what's needed)
    - Single job `publish` on `ubuntu-latest`
    - Job-level `permissions: contents: read` (checkout needs read, nothing else for NPM_TOKEN path)
    - Steps in order:
      1. `actions/checkout@v4`
      2. `pnpm/action-setup@v4` with `version: 9` (matches deploy.yml)
      3. `actions/setup-node@v4` with `node-version: 22`, `cache: pnpm`, `registry-url: https://registry.npmjs.org` (registry-url is required so setup-node writes the .npmrc that maps NODE_AUTH_TOKEN to the registry)
      4. `pnpm install --frozen-lockfile` (installs devDeps so prepack/svelte-package/publint all work)
      5. `pnpm publish --no-git-checks` — do NOT pass `--access public` here; publishConfig in package.json handles it. `--no-git-checks` is required because GitHub Actions checks out in detached HEAD state. Set env `NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}`.

    Add a comment block at the top of the workflow explaining the NPM_TOKEN setup requirement and the OIDC upgrade path:

    ```yaml
    # NPM_TOKEN setup:
    # 1. Go to npmjs.com → Access Tokens → Generate New Token (Automation type)
    # 2. Add the token as a GitHub secret named NPM_TOKEN in repo Settings → Secrets
    #
    # First publish must be done manually: pnpm publish --access public
    # After that, this workflow handles all subsequent releases.
    #
    # OIDC upgrade path (no long-lived tokens):
    # Once the package exists on npm, configure a Trusted Publisher on npmjs.com
    # (package → Settings → Publishing → Trusted Publishers), then replace
    # NODE_AUTH_TOKEN with id-token: write permission + NPM_CONFIG_PROVENANCE=true.
    ```

    Do not add a separate build step — `prepack` fires automatically before `pnpm publish` and runs the full `svelte-package && cleanup && publint` chain.
  </action>
  <verify>
