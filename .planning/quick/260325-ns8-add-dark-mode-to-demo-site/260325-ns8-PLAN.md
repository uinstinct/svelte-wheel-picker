  <action>
1. In `src/app.html`, add an inline script in `<head>` BEFORE %sveltekit.head% that:
   - Reads `localStorage.getItem('theme')` (values: 'light', 'dark', or null/absent for system)
   - If 'dark', adds class `dark` to `<html>`
   - If 'light', adds class `light` to `<html>`
   - If null (system), checks `window.matchMedia('(prefers-color-scheme: dark)').matches` and adds `dark` or `light` accordingly
   - This prevents flash-of-wrong-theme on page load

2. In `src/routes/+page.svelte`, refactor the CSS custom properties from `@media (prefers-color-scheme: dark)` to class-based:
   - Light vars stay on `:global(:root)` (the default)
   - Dark vars move to `:global(:root.dark)` instead of the media query
   - Keep the `color-scheme: light dark` on :root
   - Add `:global(:root.dark) { color-scheme: dark; }` and `:global(:root.light) { color-scheme: light; }`

3. In `src/routes/+page.svelte`, add a theme toggle in the hero section (after the h1, before the description):
   - Create a `let theme = $state<'light' | 'dark' | 'system'>('system')` initialized from `localStorage.getItem('theme') ?? 'system'` inside an `$effect` on mount (check `typeof window !== 'undefined'`)
   - Create a function `cycleTheme()` that cycles system -> light -> dark -> system
   - On theme change (via `$effect`): update `document.documentElement.classList` (remove both 'light'/'dark', add appropriate class based on theme value; for 'system', check matchMedia), and persist to localStorage (remove key for 'system')
   - Render a `<button>` with class `theme-toggle` showing current mode icon/label:
     - system: "System" (or a monitor icon via Unicode: &#x1F5A5; -- actually use plain text labels to avoid emoji: "Auto" / "Light" / "Dark")
     - light: "Light"
     - dark: "Dark"
   - Style the button: small pill-shaped, matches the install-block aesthetic (background: var(--color-surface), border-radius: 999px, padding: 4px 12px, font-size: 12px, border: 1px solid transparent, cursor: pointer, color: var(--color-text-muted))
   - Position: float right or absolute top-right of hero, or simply place before the h1 as a small aligned element. Use `position: fixed; top: 16px; right: 16px;` for consistent visibility.

4. Listen for system theme changes: add a matchMedia listener so when theme is 'system' and OS preference changes, the class updates accordingly.
