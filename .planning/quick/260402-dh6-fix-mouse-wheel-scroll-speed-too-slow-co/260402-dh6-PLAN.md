  <action>
Replace the handleWheel method in WheelPhysics class. The current implementation has two problems:
1. 100ms debounce drops most wheel events
2. Only moves +-1 item regardless of deltaY magnitude

New implementation:

```typescript
handleWheel(deltaY: number): void {
  // Cancel any in-flight animation so rapid scrolling feels immediate
  this.#cancelRaf();

  // Calculate number of items to move based on deltaY magnitude.
  // A typical mouse wheel notch sends deltaY ~100-150px.
  // Divide by itemHeight to get proportional item count, minimum 1.
  const itemsToMove = Math.max(1, Math.round(Math.abs(deltaY) / this.#itemHeight));
  const direction = deltaY > 0 ? 1 : -1;
  const steps = itemsToMove * direction;

  const rawIndex = this.#offsetToIndex(this.offset);
  const N = this.#options.length;
  const currentIndex = this.#infinite
    ? wrapIndex(rawIndex, N)
    : clampIndex(rawIndex, N);

  if (this.#infinite) {
    const next = currentIndex + steps;
    const wrapped = wrapIndex(next, N);
    const snapIndex = snapToNearestEnabled(wrapped, this.#options);
    this.animateTo(snapIndex);
  } else {
    const targetIndex = clampIndex(currentIndex + steps, N);
    const snapIndex = snapToNearestEnabled(targetIndex, this.#options);
    this.animateTo(snapIndex);
  }
}
```

Key changes:
- Remove the 100ms debounce entirely (it was the main bottleneck). The `#cancelRaf()` at the start handles rapid events by canceling the previous animation, so rapid wheel events just update the target.
- Remove the `#lastWheelTime` field usage in handleWheel (keep the field declaration in case other code references it — but it is only used here, so remove it entirely if nothing else uses it).
- Calculate `itemsToMove` from `Math.abs(deltaY) / this.#itemHeight` so a deltaY of 300 with itemHeight of 30 moves 10 items.
- Cancel in-flight animation before computing new target so rapid scrolling feels snappy.

Do NOT change startDrag, moveDrag, endDrag, animateTo, or any other method.
