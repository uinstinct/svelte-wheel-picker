## Common Pitfalls

### Pitfall 1: Confusing tsconfig.json with the effective config
**What goes wrong:** Developers look at `tsconfig.json` and don't see a `target` field, concluding it's missing or defaulting to ES3/ES5.
**Root cause:** The `"extends"` directive pulls settings from `.svelte-kit/tsconfig.json`, which is auto-generated and sets `target: "esnext"`.
**How to verify:** Run `npx tsc --showConfig` to see the merged effective configuration.

### Pitfall 2: Assuming state_invalid_placement means wrong TS target
**What goes wrong:** The `state_invalid_placement` error can occur for multiple reasons. A low TS target is one, but not the only cause.
**Other causes:** Using `$state()` outside `.svelte` or `.svelte.ts` files, using it in module-level code that gets evaluated at import time, or Svelte compiler version mismatch.
